Create a GitHub pull request for a completed feature.

Usage: /pr <FEAT-ID>
Example: /pr FEAT-003

Requires `gh` (GitHub CLI) to be installed and authenticated.
The feature must be in COMPLETE or REVIEW_PASS status.

---

## Step 1 — Load configuration

Read `.stangent/config.json`.
If it does not exist: output "Run init.py first." and stop.

Extract:
  - feature_dir    = config.paths.feature_dir
  - pr_target      = config.pipeline.pr_target_branch  (default: "main")
  - project_name   = current directory name

---

## Step 2 — Validate

If "$ARGUMENTS" is empty: output "Usage: /pr <FEAT-ID>" and stop.

Find the feature file: glob {feature_dir}/$ARGUMENTS*.md
If not found: output "Feature $ARGUMENTS not found." and stop.

Read frontmatter: title, status, branch, feature_id, slug.

Check status:
  - COMPLETE or REVIEW_PASS → proceed
  - IMPLEMENTING or REVIEWING → output "Feature is still in progress. Wait for COMPLETE." and stop.
  - PAUSED → output "Feature is paused. Resume with /resume {feature_id} first." and stop.
  - ABANDONED → output "Feature was abandoned. Cannot create PR." and stop.
  - Any other → output "Feature is not ready for PR (status: {status})." and stop.

---

## Step 3 — Check gh is available

Run: `gh --version`
If it fails: output the following and stop:
  "gh (GitHub CLI) is not installed or not authenticated.
   Install: https://cli.github.com
   Authenticate: gh auth login"

---

## Step 4 — Check branch exists and has commits

Run: `git log --oneline {pr_target}..{branch}`
If empty: output "Branch {branch} has no commits ahead of {pr_target}. Nothing to PR." and stop.

Check if branch exists on remote:
  Run: `git ls-remote --heads origin {branch}`
  If not found: ask:
    "Branch {branch} has not been pushed to origin.
     Push now? (yes/no)"
  If "yes": run `git push -u origin {branch}`
  If "no": output "Push the branch first, then re-run /pr {feature_id}." and stop.

---

## Step 5 — Build PR body

Read from the feature file:
  - ## Scope         → summary paragraph
  - ## Acceptance Criteria → checkbox list
  - ## Files Changed → list of changed files
  - ## Test Report   → coverage and status
  - ## Security Report → status line

Build PR body:

```
## Summary

{## Scope content, verbatim}

## Acceptance Criteria

{## Acceptance Criteria content, verbatim}

## Files Changed

{## Files Changed content, verbatim}

## Test & Security

- Tests: {## Test Report status and coverage delta}
- Security: {## Security Report status}

---
Feature: [{feature_id}](.stangent/features/{feature_id}-{slug}.md)
Generated by [Stangent](https://github.com/yourname/stangent)
```

---

## Step 6 — Confirm

Output a preview:
  "PR to create:
   Title: {feature_id}: {title}
   From:  {branch}
   Into:  {pr_target}
   Body:  (shown above)

   Create PR? (yes/no)"

Wait for response. If not "yes": output "PR creation cancelled." and stop.

---

## Step 7 — Create PR

Run:
```
gh pr create \
  --title "{feature_id}: {title}" \
  --body "{pr_body}" \
  --base {pr_target} \
  --head {branch}
```

If the command succeeds: output the PR URL returned by gh.
If it fails: output the error from gh and stop.

---

## Step 8 — Update feature file

Append to ## Pipeline History:
  `[timestamp] | PR_CREATED | developer | {pr_url}`

Output:
  ```
  ✓ PR created: {pr_url}
  Branch: {branch} → {pr_target}
  ```
