# Supabase RLS Policies

## Always enable RLS first

```sql
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
-- Without this, all policies are ignored and table is fully public to authenticated users
```

## auth.uid() performance trap — most critical optimization

```sql
-- WRONG: auth.uid() re-evaluated for every row
CREATE POLICY "Users own their posts"
ON posts FOR ALL TO authenticated
USING (auth.uid() = user_id);

-- CORRECT: evaluated once per query (wrap in select)
CREATE POLICY "Users own their posts"
ON posts FOR ALL TO authenticated
USING ((select auth.uid()) = user_id);
```

Same applies to `auth.role()`, `auth.jwt()`, `current_setting()`.

## INSERT requires WITH CHECK, not USING

```sql
-- WRONG: USING has no effect on INSERT
CREATE POLICY "Users insert own posts"
ON posts FOR INSERT TO authenticated
USING (user_id = (select auth.uid()));

-- CORRECT: WITH CHECK validates incoming row data on INSERT
CREATE POLICY "Users insert own posts"
ON posts FOR INSERT TO authenticated
WITH CHECK (user_id = (select auth.uid()));

-- UPDATE: needs both USING (which rows) and WITH CHECK (what values allowed)
CREATE POLICY "Users update own posts"
ON posts FOR UPDATE TO authenticated
USING ((select auth.uid()) = user_id)
WITH CHECK ((select auth.uid()) = user_id);
```

## AI-generated insecure default — do not use

```sql
-- GENERATED BY AI — allows everyone to read everything
CREATE POLICY "Enable read access for all users"
ON posts FOR SELECT USING (true);

-- CORRECT: restrict to authenticated unless data is truly public
CREATE POLICY "Authenticated users can read posts"
ON posts FOR SELECT TO authenticated USING (true);
```

## Multi-tenant pattern

```sql
CREATE POLICY "Tenant isolation"
ON projects FOR ALL TO authenticated
USING (org_id::TEXT = auth.jwt()->>'org_id')
WITH CHECK (org_id::TEXT = auth.jwt()->>'org_id');
```

## Membership subquery

```sql
CREATE POLICY "Team members access documents"
ON documents FOR SELECT TO authenticated
USING (
  team_id IN (
    SELECT team_id FROM team_members
    WHERE user_id = (select auth.uid())
  )
);
```

## Testing RLS (SQL Editor bypasses RLS — simulate instead)

```sql
SELECT set_config('request.jwt.claims', json_build_object(
  'sub', 'your-test-user-uuid',
  'role', 'authenticated',
  'org_id', 'your-org-uuid'
)::TEXT, true);

SET ROLE authenticated;
SELECT * FROM projects;  -- should only return rows this user owns
RESET ROLE;
```

## Always index policy columns

```sql
CREATE INDEX idx_posts_user_id ON posts(user_id);
CREATE INDEX idx_documents_team_id ON documents(team_id);
```

## Views bypass RLS — fix with security_invoker (PostgreSQL 15+)

```sql
CREATE VIEW public_posts
WITH (security_invoker = true) AS
SELECT * FROM posts WHERE published = true;
```

## Mistake cheatsheet

| Mistake | Fix |
|---|---|
| `USING (true)` on all rows | Add `TO authenticated` and real conditions |
| No `ENABLE ROW LEVEL SECURITY` | Always include this line |
| `auth.uid()` without `(select ...)` | `(select auth.uid())` |
| INSERT policy uses `USING` | Use `WITH CHECK` |
| Testing in Supabase SQL Editor | Use `set_config` + `SET ROLE` |
| Missing index on `user_id` / `org_id` | Add btree indexes |
| `service_role` key in frontend | Server-side only |
