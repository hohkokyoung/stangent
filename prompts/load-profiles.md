## Loading Language Profiles

This procedure is used by all pipeline agents. Follow these steps exactly.

### Step 1 — Extract from config

From `.stangent/config.json` extract:
- `config.profiles`       — list of active profile names; `profiles[0]` is the primary
- `config.profile_roots`  — `{name: src_root}` map

Derive: `project_root = Path(config_path).parent.parent`

### Step 2 — Read each profile

For each name in `config.profiles`:
  Read `{project_root}/.stangent/profiles/{name}.md` → store as `profiles[name]`
  If the file does not exist: stop immediately and output:
  > Profile '{name}' not found. Re-run: python init.py --profile <valid-profile>

  Return FAILED.

### Step 3 — File-to-profile routing

When you need to select a profile for a specific file path:
- Check `config.profile_roots` — use the profile whose `src_root` the file path starts with
- If no match or ambiguous: fall back to `config.profiles[0]` (primary profile)

### Step 4 — Project-wide aggregation

For settings used before any specific file is known (e.g. tree scan):
- `anchor_files`: combine lists from all profiles
- `exclude_dirs`: union of all profiles' exclude_dirs

### Step 5 — Pass 3 file read limits

When doing a targeted Pass 3 scan (reading specific files for a feature), enforce:

**Hard limits:**
- Maximum 15 files per Pass 3 run. If more files are candidates, prioritise by relevance to the feature task. Log skipped files to the Run Log.
- Files over 300 lines: read only the first 100 lines, then use grep to locate the specific sections relevant to the task (read ±10 lines around each match). Do not read the full file.

**Skip list — never read these in any pass:**
- Lock files: `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `Pipfile.lock`, `poetry.lock`, `Cargo.lock`, `composer.lock`, `go.sum`
- Generated files: any file whose first line contains `// Code generated`, `# Code generated`, `// DO NOT EDIT`, `# DO NOT EDIT`, `// AUTO-GENERATED`, `// autogenerated`
- Build artefacts: files under `dist/`, `build/`, `out/`, `target/`, `.next/`, `__pycache__/`, `.pytest_cache/`, `node_modules/`, `.tox/`, `coverage/`
- Binary files: `.png`, `.jpg`, `.jpeg`, `.gif`, `.ico`, `.woff`, `.woff2`, `.ttf`, `.eot`, `.pdf`, `.zip`, `.tar`, `.gz`, `.exe`, `.dll`, `.so`, `.dylib`, `.pyc`, `.pyo`
